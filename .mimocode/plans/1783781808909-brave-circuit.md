# Plan: Fix notification settings save on client site

## Problem
When user clicks "Сохранить" in notification settings:
1. Browser permission request for notifications appears
2. Save button does nothing - settings not saved
3. No errors in console

## Root Cause
The `buildPatch` function in `bindProfileForm(sheetRoot, "profile-notifications-form", ...)` (line 6714) calls `ensureHomePushSubscription({ forcePrompt: true })` when any push notification is enabled. This:
- Requests browser notification permission
- If permission denied, throws error that closes the sheet
- User wants permission requested at booking time, not in settings

## Solution

### 1. Remove push subscription logic from notification settings save
**File: `client-app.js` (line 6714-6741)**

Simplify the `buildPatch` callback to just return the checkbox values without requesting permission:

```javascript
bindProfileForm(sheetRoot, "profile-notifications-form", async (formData) => {
  return {
    bookingNotificationsEnabled: formData.get("bookingNotificationsEnabled") === "on",
    bookingPushNotificationsEnabled: formData.get("bookingPushNotificationsEnabled") === "on",
    balanceNotificationsEnabled: formData.get("balanceNotificationsEnabled") === "on",
    balancePushNotificationsEnabled: formData.get("balancePushNotificationsEnabled") === "on",
  };
});
```

### 2. Request notification permission when making a booking
**File: `client-app.js` (line ~5036, in `submitBooking`)**

Add permission request at the start of `submitBooking()`:

```javascript
const submitBooking = async () => {
  // Request notification permission early if push is enabled
  const user = state.payload?.user || {};
  if (user.bookingPushNotificationsEnabled !== false || user.balancePushNotificationsEnabled !== false) {
    ensureHomePushSubscription({ forcePrompt: true }).catch(() => {});
  }
  // ... rest of existing code
};
```

### 3. Request notification permission on app load (when push is enabled)
**File: `client-app.js` (line ~6980, in bootstrap)**

The existing code at line 6986 already does this:
```javascript
if (getActivePushToggleState(state.payload?.user || {})) {
  ensureHomePushSubscription({ forcePrompt: false }).catch(() => null);
}
```
Keep this as-is.
