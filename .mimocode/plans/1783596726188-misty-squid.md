# Plan: Creator Income Page + Backup Retention

## Goal
1. Add an "Доходы" (Income) page for the creator role to track payments from businesses
2. Change backup retention from 30 days to 10 days

---

## Part 1: Creator Income Page

### 1.1 Database Model — `CreatorIncome`

**File:** `prisma/schema.postgresql.prisma`

Add a new model to the **global (public) schema** (not tenant — income is cross-business):

```prisma
model CreatorIncome {
  id           String   @id
  businessId   String
  amount       Float
  periodStart  String?  // "2026-07-01"
  periodEnd    String?  // "2026-07-31"
  date         String   // payment date "2026-07-05"
  note         String?
  createdAt    DateTime @default(now())
  updatedAt    DateTime @updatedAt

  business Businesses @relation(fields: [businessId], references: [id], onDelete: Cascade)

  @@index([businessId])
  @@index([date])
  @@map("CreatorIncome")
}
```

Also add `creatorIncome CreatorIncome[]` relation to the `Businesses` model.

**Migration:** `prisma migrate dev --name add_creator_income`

### 1.2 Backend API — Creator Income Routes

**New file:** `routes/creatorIncomeRoutes.js`

Endpoints (all require `creator` role):

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/api/creator/income` | List all income records (supports `?businessId=` and `?from=` / `?to=` query params) |
| `POST` | `/api/creator/income` | Create income record |
| `PUT` | `/api/creator/income/:id` | Update income record |
| `DELETE` | `/api/creator/income/:id` | Delete income record |
| `GET` | `/api/creator/income/summary` | Aggregated summary: total by business, total by month |

**Wire in:** `server-source/21-domain-services.js` — add `creatorIncomeRoutes` to the service wiring
**Register in:** `server-source/30-runtime-api.js` — call `registerCreatorIncomeRoutes()`

### 1.3 Frontend — Creator Income View

**New file:** `crm-source/34-creator-income.js`

Components:
- **`CreatorIncomeView`** — main page component
  - Date range filter (preset buttons: this month, last 3 months, this year, all time)
  - Business filter dropdown (populated from `/api/creator/businesses`)
  - Summary cards: total income, avg monthly, this month
  - Monthly bar chart (simple CSS-based, no chart library needed)
  - Income records table with columns: Date, Business, Amount, Period, Note, Actions
  - Create/Edit modal for income records
- **`IncomeCreateModal`** — form: business (select), amount, date, period start/end, note
- **`IncomeChart`** — simple monthly totals bar chart

**Style:** Follow existing CRM patterns — `crm-soft-panel`, `crm-soft-card`, `crm-ghost-btn`, `crm-action-btn`

### 1.4 Tab Registration for Creator

**File:** `crm-source/00-core-config.js`

Update `VIEW_TABS_BY_ROLE[ROLE_CREATOR]`:
