# План: Drag-and-drop reorder для должностей + creator revenue page + backup retention

## 1. Positions reorder (drag-and-drop)

### Сервер
- Добавить `POST /api/Positions/reorder` в `routes/adminCrudRoutes.js`
- Принимает `{ orderedIds }`, обновляет `orderIndex` для каждой позиции
- Паттерн аналогичен `POST /api/services/full/reorder`

### Клиент
- Добавить `handleReorderPositions` в `60-auth-app.js`
- Добавить `positionReorderBusy` state
- Передать `onReorderPositions` и `positionReorderBusy` через `42-tables-workspace.js` в `PositionsView`
- В `32-revenue-view.js` (PositionsView) добавить drag-and-drop логику:
  - Pointer Events API (как в ServicesView)
  - Drag state, refs, FLIP animations
  - `buildReorderedIds` для определения позиции вставки
  - Автоматическая нумерация уровней (orderIndex + 1)
