# Plan: Fix double animation on shop page

## Problem
When opening the shop page on the client site, there's a visible double animation — the whole page slides up AND each child element slides up independently, creating a redundant visual effect.

## Root Cause
Two independent CSS animations fire simultaneously on the shop page:

1. **`.shop-page` container** (`client-app.css:7672-7674`): `animation: page-enter 380ms` — animates the `<section>` itself (opacity 0→1, translateY 10px→0)

2. **Staggered children** (`client-app.css:2001-2024`): `.route-transition .page:not(.booking-page) > *` — animates every direct child (opacity 0→1, translateY 18px→0 with staggered delays)

Both match when `state.routeTransitionActive = true` injects `route-transition` class on the app shell during navigation. The `.page.shop-page` class isn't excluded from the children animation selector (only `.booking-page` is excluded).

## Fix
Remove the `page-enter` animation from `.shop-page` — it already gets the staggered children animation from `.route-transition`, which looks better for a multi-element page.

### File: `client-app.css` (line 7672-7674)

**Remove:**
```css
.shop-page {
  animation: page-enter 380ms cubic-bezier(.2,.8,.2,1) both;
}
```

Keep the `animation: none` override on `.shop-cart-toggle` (line 7675-7676) since it prevents the cart button from animating.
