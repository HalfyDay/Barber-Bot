# Plan: 4 CRM/client-site improvements

## 1. Phone input: allow 7 and 8 as first local digit

### Problem
When user types `7` as first digit, `formatPhoneDisplay` (CRM) and `extractPhoneDigits` (client) treat it as country code and strip it. User can't type a local number starting with `7` or `8`.

### Fix
**File: `crm-source/01-domain-utils.js`**
- `formatPhoneDisplay` (line 698): Remove the `else if (!normalized.startsWith('7'))` branch that auto-prepends `7`. Only convert `8→7` when the string is exactly 11 digits (old format). For shorter input, don't auto-prepend.
- Same logic in `crm-source/60-auth-app.js` (line 68): identical `formatPhoneDisplay` — apply same fix.

**File: `client-app.js`**
- `extractPhoneDigits` (line 1330): Remove the `digits.startsWith("7")` from the if condition. Only strip leading `8` (old format conversion). `7` should be kept as a local digit.
- The display always shows `+7 (XXX) XXX-XX-XX` — the country code `+7` is hardcoded in the output.

### Files
| File | Line | Change |
|------|------|--------|
| `crm-source/01-domain-utils.js` | 698-718 | Fix `formatPhoneDisplay` — don't auto-prepend `7` |
| `crm-source/60-auth-app.js` | 68-88 | Same `formatPhoneDisplay` fix |
| `client-app.js` | 1330-1336 | Fix `extractPhoneDigits` — don't strip leading `7` |
