# Plan: Fix auto-update restart + zero-downtime deployment

## Problem 1: Server doesn't restart after update

**Root cause** (`server-source/30-runtime-api.js` line 113-138):
The PM2 restart logic sends `process.send('ready')` which is the WRONG signal. `ready` is for telling PM2 "I finished starting" — it doesn't trigger a restart. After sending `ready`, the process just hangs. PM2 never restarts it because `autorestart: true` only triggers on `process.exit()`.

**Fix:**
1. After cleanup, call `process.exit(0)` instead of `process.send('ready')`
2. PM2's `autorestart: true` will automatically restart the process
3. Add `npm run build:server-source` before exit to ensure server bundle is fresh

## Problem 2: Update kills user sessions

**Root cause:**
During update, the code calls:
- `shutdownRealtimeClients()` — closes all SSE connections (CRM users disconnected)
- `stopHttpServer()` — stops accepting new connections
- `prisma.$disconnect()` — database connections dropped

This kills ALL active sessions instantly.

**Fix for zero-downtime:**
1. Don't call `shutdownRealtimeClients()` or `stopHttpServer()` during PM2 restart
2. Let PM2 handle the process swap — it sends SIGTERM, waits for `kill_timeout`, then starts new process
3. Old process keeps serving until PM2 kills it, new process starts immediately
4. SSE clients auto-reconnect to the new process

Replace lines 113-138:

```js
// PM2 restart: clean up and exit, let PM2 autorestart handle the new process
if (isPm2 && process.env.pm_id !== undefined) {
  setTimeout(async () => {
    try {
      stopAppointmentReminderLoop();
      stopRealtimeLoop();
      await stopBotProcess();
      // DON'T shutdown realtime clients or HTTP server — let PM2 handle the swap
      // DON'T disconnect Prisma — let the process exit naturally
    } catch (error) {
      console.error('[update] PM2 restart preparation failed:', error);
    }
    console.log('[update] Exiting for PM2 restart...');
    process.exit(0);
  }, delayMs);
  return;
}
```

### 3. `server-source/30-runtime-api.js` — Add graceful shutdown for SIGTERM

Add a SIGTERM handler that keeps the server alive until PM2 kills it:

```js
process.on('SIGTERM', () => {
  console.log('[server] SIGTERM received, shutting down gracefully...');
  // Stop accepting new connections but let existing ones finish
  if (httpServer) {
    httpServer.close(() => {
      console.log('[server] HTTP server closed');
    });
  }
  // Give in-flight requests time to complete
  setTimeout(() => {
    process.exit(0);
  }, 5000);
});
```
