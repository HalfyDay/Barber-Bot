# Анализ: Автообновление и магазин

## Что проверено

Система автообновления (`applyUpdate` в `services/updateManager.js`) делает:
1. `git pull --ff-only` — забирает новый код
2. `npm install` — если изменился package.json
3. Prisma миграции — если изменился schema
4. `runPostUpdateDatabaseFixes()` — **всегда** запускается, патчит tenant-схемы
5. `npm run build:web` — **только** если изменились файлы из списка

## Проблема: `shouldRebuildWebForChangedFile`

Триггеры пересборки:
- `crm-source/` файлы ✅ (CRM соберётся)
- `styles.tailwind.css` ✅
- `script.js`, `index.html`, etc. ✅

**Не触发 пересборки:**
- `client-app.js` — главный файл клиента (магазин)
- `client-app.css` — стили клиента (магазин)
- `server-source/` — серверный бандл
- `routes/` — новые роуты
- `services/` — новые сервисы

## Что нужно исправить

Добавить в `shouldRebuildWebForChangedFile`:
```js
file === 'client-app.js' ||
file === 'client-app.css' ||
file.startsWith('server-source/') ||
file.startsWith('routes/') ||
file.startsWith('services/')
```

Также нужно добавить `npm run build:server-source` в `applyUpdate` при изменении `server-source/` или `routes/`.

## Итого

**Да, нужны изменения.** Без них автообновление не соберёт:
- Клиентские файлы магазина (`client-app.js`, `client-app.css`)
- Серверные файлы магазина (`routes/shopRoutes.js`, `services/shopService.js`)

Prisma и tenant-схемы обновятся автоматически (благодаря `runPostUpdateDatabaseFixes`).
